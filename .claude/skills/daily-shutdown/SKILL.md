# Daily Shutdown

Reference: your team's daily cadence (engineering playbook) + your SOP for time tracking.

## Routine (~10 minutes)

### 1. Review today's todos
- What's done? → tick it off.
- What's still open? → move it to tomorrow, or flag it if the team needs to know.

### 2. Time log
Record time per project for today:
```
Project A: X h | Project B: X h | Hiring/Docs: X h | Admin: X h
```
Add it under the "Time Log" section of the daily note.

> Glossary (TH): บันทึกเวลาต่อ project ลงใน daily note.

### 3. Capture learnings
If you learned something today → capture it into `{{VAULT}}/inbox/fleeting/` (use the `capture-knowledge` skill).

### 4. PR queue check
- Any PR still waiting on your review?
- If it's over the 1-day SLA → tell the team you'll review it first thing tomorrow.

### 5. Set tomorrow's intent
Write it briefly in the daily note:
> "Tomorrow: [the 3 most important things]"

### 6. Stop time tracking
Stop your time tracker / timer.
